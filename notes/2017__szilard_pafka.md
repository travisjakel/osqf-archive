## Key findings
- High-performance packages (e.g., data.table, xgboost) are critical for big data analysis.  
  > Use R/Python and high performance packages (e.g. data.table, xgboost)
- Random forest on 1% of data can outperform linear models on full datasets.  
  > random forest on 1% of data beats linear on all data
- Distributed computing should only be used for embarrassingly parallel tasks.  
  > Only distribute embarrassingly parallel tasks (e.g. hyperparameter search for machine learning)
