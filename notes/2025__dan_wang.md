## Key findings
- Backtesting requires defining investment rules and applying them to historical data.  
  > Define a set of investment rules and processes, key parameters, and apply those rules to historical data.
- Cloud computing offers pay-as-you-go pricing and scalability for backtesting.  
  > Pay-as-you-go: no fixed costs. Scalability: Large clouds providers have functions distributed over multiple locations...
- In-sample testing can cause overfitting and survival bias in backtesting.  
  > In-sample testing: Using the same data for both training and testing can inflate performance due to over-fitting. Survival bias: Index content changes over time...
- Azure Storage Explorer is emphasized as a key tool for cloud backtesting.  
  > Microsoft Azure Storage Explorer (repeated 50+ times in the deck)
- Performance metrics include Sharpe ratio, drawdowns, and correlation analysis.  
  > Performance metrics for back-testing: Net PnL/Sharpe, Risk adjusted returns, Correlation between different strategies
